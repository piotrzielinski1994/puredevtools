- 2026-07-10: A visually-hidden (`sr-only`) checkbox with the visible track/thumb as *sibling* spans is not clickable - a real mouse click on the visuals never reaches the input, so `onChange` never fires. Wrap the input + spans in a `<label>` so clicks forward to the input. Testing-library `fireEvent.click(input)` hits the input directly and hides this bug; test the label click (`getByLabelText(...).closest('label')`) to catch it. Affected `Switch` (global + per-rule toggles).
- 2026-07-10: You cannot override a **main-frame document navigation** (URL typed in the address bar) with a `fetch`/`XHR` page-layer patch - the top-level HTML loads before any script runs. Overriding it on Chrome needs `chrome.debugger` (yellow "being debugged" banner) with no cross-browser equivalent. Peer tools (MockExpress, Requestly page layer) don't do it either; they override `fetch`/`XHR` the page's JS makes. Test overrides by calling `fetch(url)` from the console, not by loading the URL as a page.
- 2026-07-10: `zod.object({...})` is non-strict by default - it silently strips unknown keys, so a stored/imported rule carrying a removed matcher field (e.g. `resourceTypes`) would pass validation. Use `.strict()` on `matchers` when the point is to *reject* legacy shapes at the import boundary.
- 2026-07-12: The sidebar folder tree ports requi's DnD with `@dnd-kit/core` **core primitives only** (`useDraggable` + `useDroppable` per row + manual pointer-Y drop projection in `tree-locate.ts`), NOT `@dnd-kit/sortable`'s `SortableContext`/`useSortable`. `SortableContext` is for a flat sortable list; a nested tree with reparenting needs the low-level primitives. Don't add `@dnd-kit/sortable` for the tree.
- 2026-07-12: The engine consumes rules purely as an **ordered array** (`repository.getAll()` -> `decide.find()` first-match) and never reads a numeric ordering field. That's why folders could be added as a storage/UI tree that `flatten`s (DFS pre-order) to `Rule[]` without touching `engine/**`, `content/bridge.ts`, `match.ts`, `decide.ts`. When adding structure over rules, preserve the flat ordered `getAll()` contract and the engine stays untouched.
- 2026-07-12: Simulating `@dnd-kit` PointerSensor drags in jsdom needs three shims/moves: (1) a `PointerEvent` shim that sets `isPrimary: true` (the activator rejects `!event.isPrimary`), (2) `Element.prototype.hasPointerCapture`/`setPointerCapture`/`releasePointerCapture` stubs, (3) stub `getBoundingClientRect` on **every** droppable row BEFORE the drag (dnd-kit measures all rects at drag start; `pointerWithin` finds no target otherwise). Drive it as pointerDown(source) -> pointerMove(document, past 5px) -> pointerMove(target) -> pointerMove(document) -> pointerUp(document). See `src/ui/test-setup.ts` + `SidebarTree.test.tsx`.
